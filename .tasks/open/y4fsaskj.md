# FreeBSD pkg packaging for epithet

## Context

epithet already cross-compiles for FreeBSD (amd64, arm64) via goreleaser. The goal is to create proper FreeBSD `.pkg` binary packages and host a package repository so FreeBSD servers can run `pkg install epithet`.

Since you have FreeBSD servers, we'll use native `pkg create` and `pkg repo` directly — no cross-platform workarounds needed.

## Background: what's in a FreeBSD pkg

A `.pkg` file is a tar archive (zstd-compressed by default) containing:
1. `+COMPACT_MANIFEST` — JSON metadata for the repo catalog
2. `+MANIFEST` — full JSON metadata with file checksums and install scripts
3. The payload files at their install paths

A repository is static files served over HTTP: `meta.conf`, `packagesite.txz` (catalog), and the `.pkg` files themselves. Generated by `pkg repo <dir>`.

## Files to create

### `packaging/freebsd/pkg-manifest.ucl`

Package metadata template with `%%VERSION%%` and `%%ARCH%%` placeholders:

```ucl
name: "epithet"
version: "%%VERSION%%"
origin: "security/epithet"
comment: "SSH certificate authority with short-lived certificates"
arch: "FreeBSD:*:%%ARCH%%"
www: "https://github.com/epithet-ssh/epithet"
maintainer: "brianm@apache.org"
prefix: /usr/local
desc: <<EOD
SSH certificate authority that replaces key-based authentication with
short-lived certificates. Supports per-connection SSH agents, OIDC
authentication, and real-time policy enforcement.
EOD
categories: ["security"]
```

### `packaging/freebsd/pkg-plist`

```
bin/epithet
etc/rc.d/epithet_server
etc/rc.d/epithet_agent
share/examples/epithet/epithet.config.example
share/examples/epithet/sshd_config.d/100-epithet.conf
share/examples/epithet/bash_auth_example.bash
share/examples/epithet/policy.example.yaml
```

Note: paths are relative to `prefix` (`/usr/local`).

### `packaging/freebsd/rc.d/epithet_server`

FreeBSD rc.d script for `epithet server` (combined CA + policy). Follows the standard rc.d pattern:

```sh
#!/bin/sh
# PROVIDE: epithet_server
# REQUIRE: NETWORKING
# KEYWORD: shutdown

. /etc/rc.subr

name="epithet_server"
rcvar="epithet_server_enable"
command="/usr/local/bin/epithet"
command_args="server ${epithet_server_flags}"
pidfile="/var/run/${name}.pid"
start_cmd="${name}_start"

epithet_server_start() {
    /usr/sbin/daemon -p ${pidfile} -u epithet \
        ${command} ${command_args}
}

load_rc_config $name
: ${epithet_server_enable:="NO"}
run_rc_command "$1"
```

A similar `epithet_agent` script for the broker/agent daemon.

### `packaging/freebsd/build-pkg.sh`

Shell script that runs on FreeBSD. Takes version, arch, and path to the binary as args:

1. Creates staging directory with proper layout under `/usr/local/`
2. Copies binary, rc.d scripts, example files
3. Substitutes `%%VERSION%%` and `%%ARCH%%` in the manifest
4. Runs `pkg create -M manifest -r staging/ -p plist -o output/`

### `packaging/freebsd/repo-update.sh`

Runs on the FreeBSD server that hosts the repository:

1. Copies `.pkg` files into repo directory (organized by ABI, e.g., `FreeBSD:14:amd64/`)
2. Runs `pkg repo <dir>` to regenerate `packagesite.txz` and `meta.conf`
3. Optionally signs with RSA key if `--sign` flag given

### `packaging/freebsd/README.md`

Setup instructions covering:
- How to build a package from a goreleaser tarball
- How to set up the repository on a FreeBSD server
- How to configure client machines to use the repo
- Optional: signing the repository

## Repository hosting

The repo is just static files. Two good options:

**Option A — Local web server** (simplest to start):
```
/var/www/pkg/epithet/
├── FreeBSD:14:amd64/
│   ├── All/
│   │   └── epithet-0.1.0.pkg
│   ├── meta.conf
│   └── packagesite.txz
└── FreeBSD:14:arm64/
    ├── All/
    │   └── epithet-0.1.0.pkg
    ├── meta.conf
    └── packagesite.txz
```

Serve with nginx/apache. Clients configure:
```
epithet: { url: "https://pkg.example.com/epithet/${ABI}" }
```

**Option B — S3/R2**: Same directory structure, synced with `aws s3 sync` or `rclone`. Works because it's all static files. Slightly more work to set up initially but zero server maintenance.

Both options work; `repo-update.sh` will generate the repo locally on FreeBSD, then you rsync/s3-sync it wherever.

## Workflow summary

```
goreleaser (any OS)           FreeBSD server
───────────────               ──────────────
epithet_X.Y.Z_freebsd_*.tar.gz
       │
       └──── scp ──────────> build-pkg.sh 0.1.0 amd64 ./epithet
                              build-pkg.sh 0.1.0 arm64 ./epithet
                              repo-update.sh /var/www/pkg/epithet
                                    │
                              serve via nginx / sync to S3
```

## Verification

1. On a FreeBSD server, run `build-pkg.sh` with a test binary
2. Inspect: `pkg info -F epithet-*.pkg` — verify metadata and file list
3. Install locally: `pkg add epithet-*.pkg`
4. Verify binary: `epithet --version` and `epithet config-doc`
5. Verify rc.d: `service epithet_server onestart` (with minimal config)
6. Generate repo: `repo-update.sh /tmp/test-repo`
7. On another FreeBSD machine, add the repo config and run `pkg install epithet`
